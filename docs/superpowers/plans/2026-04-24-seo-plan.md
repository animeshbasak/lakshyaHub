# Lakshya — SEO Plan

> **Scope:** executable, phased SEO strategy for Lakshya (Next.js 16 + React 19 + Supabase). Each phase is a self-contained checklist with files, commands, and verification. Phases map to the integration roadmap (`2026-04-24-careerops-integration.md`).

> **North Star:** every user journey — "AI job evaluator," "how to land Head of Applied AI," "Teal vs Lakshya" — has a dedicated page that ranks, converts, and credits career-ops (santifer). No page indexable by Google should 404 for a bot.

---

## Strategic Context

Lakshya sells to a sophisticated segment (senior ICs, mid-level PMs, career-switchers) who research tools before paying $19–$49/mo. Those people enter at long-tail queries ("best AI job evaluator", "what does a Forward Deployed Engineer do", "how to land agentic AI role"). Paid ads at those intents are expensive and competitive; organic content is compounding and defensible.

Three compounding SEO assets drive unit economics:

1. **Archetype guides** (6 pillars × 2000–3000 words) — rank for "how to get into X role", convert via in-content CTA.
2. **Programmatic `/share/:evalId` pages** — every user who opts to publish a report is a backlink, a conversion touchpoint, and a long-tail indexable URL.
3. **`/compare/:competitor` pages** — intercept bottom-funnel "Teal vs X" searches where intent is already commercial.

Current state: the landing page is a `'use client'` component with no per-route metadata, no sitemap, no structured data, no OG images. Every phase below is additive — we are building the SEO stack from zero.

---

## Non-negotiables

1. Every public page must export `metadata` or `generateMetadata` with title + description + canonical. RSC first, `'use client'` only below the metadata boundary.
2. `/app/*` and `/api/*` must be `noindex, nofollow` in `robots.ts` and per-route metadata.
3. Every OG image must be dynamic via `ImageResponse` from `next/og` (not `@vercel/og` — wrong package for Next.js 16).
4. No programmatic page with < 60% text uniqueness vs its sibling — fallback to `noindex` if thin.
5. No competitor-disparaging copy on `/compare/*`. Stick to capability diffs the user can verify.
6. Never promise a ranking — describe strategy + measurement. Track lift quarterly.

## Deferred

- International SEO (DE/FR/JA/PT/RU — career-ops modes exist; launch after English stabilizes at $10k MRR)
- Video SEO (YouTube)
- Paid retargeting (separate channel)

---

## Next.js 16 SEO facts (verified)

Before writing metadata code, verify against `node_modules/next/dist/docs/`:

- **`params` is a Promise** — page/layout/route signatures must `await params` in async handlers.
- **`'use cache'` replaces `unstable_cache`** — use the new directive for ISR-like revalidation.
- **`ImageResponse` ships in `next/og`** — `import { ImageResponse } from 'next/og'`, not `@vercel/og`.
- **Metadata files (`app/icon.tsx`, `app/apple-icon.tsx`, `app/opengraph-image.tsx`, `app/twitter-image.tsx`) are special file conventions** — Next.js wires them into layouts automatically.
- **`sitemap.ts` and `robots.ts`** are the conventional routes for those assets.

```typescript
// src/app/sitemap.ts
import type { MetadataRoute } from 'next'
import { createServiceClient } from '@/lib/supabase/service'
import { ARCHETYPES } from '@/lib/careerops/archetypes'

const BASE = 'https://lakshya.app'

export default async function sitemap(): Promise<MetadataRoute.Sitemap> {
  const statics: MetadataRoute.Sitemap = [
    { url: `${BASE}/`,         lastModified: new Date(), changeFrequency: 'weekly',  priority: 1.0 },
    { url: `${BASE}/pricing`,  lastModified: new Date(), changeFrequency: 'monthly', priority: 0.9 },
    { url: `${BASE}/about`,    lastModified: new Date(), changeFrequency: 'yearly',  priority: 0.6 },
  ]

  const guides: MetadataRoute.Sitemap = ARCHETYPES.map(a => ({
    url: `${BASE}/guides/${a}`,
    lastModified: new Date(),
    changeFrequency: 'monthly',
    priority: 0.85,
  }))

  const supabase = createServiceClient()
  // Public shares (only those flagged public = true)
  const { data: shares } = await supabase
    .from('evaluations')
    .select('id, updated_at')
    .eq('is_public', true)
    .limit(5000)

  const shareEntries: MetadataRoute.Sitemap = (shares ?? []).map(s => ({
    url: `${BASE}/share/${s.id}`,
    lastModified: new Date(s.updated_at),
    changeFrequency: 'yearly',
    priority: 0.5,
  }))

  return [...statics, ...guides, ...shareEntries]
}
```

```typescript
import { notFound } from 'next/navigation'
import { ARCHETYPES, type Archetype } from '@/lib/careerops/archetypes'
import { getGuide } from '@/lib/content/guides'
import type { Metadata } from 'next'

export async function generateStaticParams() {
  return ARCHETYPES.map(a => ({ archetype: a }))
}

export async function generateMetadata({ params }: { params: Promise<{ archetype: string }> }): Promise<Metadata> {
  const { archetype } = await params
  if (!ARCHETYPES.includes(archetype as Archetype)) return {}
  const guide = await getGuide(archetype as Archetype)
  return {
    title: guide.title,
    description: guide.metaDescription,
    alternates: { canonical: `https://lakshya.app/guides/${archetype}` },
    openGraph: {
      title: guide.title,
      description: guide.metaDescription,
      images: [{ url: `/og?page=guide&archetype=${archetype}`, width: 1200, height: 630 }],
    },
  }
}

export default async function GuidePage({ params }: { params: Promise<{ archetype: string }> }) {
  const { archetype } = await params
  if (!ARCHETYPES.includes(archetype as Archetype)) notFound()
  const guide = await getGuide(archetype as Archetype)
  return <GuideContent guide={guide} />
}
```
